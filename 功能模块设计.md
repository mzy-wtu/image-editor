3.3 功能模块设计

依据需求分析阶段确立的系统边界与用户角色，本系统采用前后端分离的架构模式，将整体功能按业务职责划分为用户模块、管理员模块、图像生成模块、历史记录模块以及前端页面模块。各模块通过定义清晰的 RESTful 接口进行数据交互，前端基于 Vue.js 构建单页面应用，后端以 Flask 框架承载业务逻辑，并借助 Flask-Login 实现统一的身份认证与权限校验。模块间的低耦合设计使得系统具备良好的可维护性和扩展性，下面对各模块的详细设计进行阐述。

3.3.1 用户模块

用户模块是系统的访问入口，承担用户身份注册、登录认证、会话维持及状态查询等基础功能。该模块通过哈希算法（pbkdf2）对密码进行单向加密存储，确保用户凭证安全。登录成功后，服务器端创建持久会话并写入加密 Cookie，后续请求由 login_required 装饰器自动验证，从而为其他业务模块提供统一的身份屏障。模块同时预留了 is_admin 字段，用于区分普通用户与管理员权限，为管理员模块的权限判定提供依据。表3-1列出了用户模块的核心接口。

表3-1 用户模块接口

| 功能 | 端点 | 方法 | 说明 |
|------|------|------|------|
| 注册 | /api/register | POST | 接受用户名、密码、邮箱，创建账户 |
| 登录 | /api/login | POST | 验证用户名与密码，返回会话凭证 |
| 登出 | /api/logout | GET | 清除当前会话，退出登录 |
| 状态查询 | /api/login | GET | 返回当前用户的登录状态信息 |

3.3.2 管理员模块

管理员模块面向系统管理者，提供全局用户管理与数据审计功能。所有接口均需通过管理员权限校验，普通用户无法访问。该模块支持以列表形式获取全部注册用户的基本信息，并可对指定用户执行启用或禁用操作，实现对系统使用权限的动态控制。此外，管理员能够查看任一用户的图像生成历史，以便进行内容审查或使用情况分析。前端管理界面集成了统计卡片与分页表格，直观展示总用户数、活跃状态等摘要信息。接口设计如表3-2所示。

表3-2 管理员模块接口

| 功能 | 端点 | 方法 | 说明 |
|------|------|------|------|
| 用户列表 | /api/admin/users | GET | 获取所有用户信息列表 |
| 修改用户 | /api/admin/users/<id> | PUT | 禁止或启用指定用户账号 |
| 用户图像 | /api/admin/users/<id>/images | GET | 查看指定用户的全部生成历史（分页） |

3.3.3 图像生成模块

图像生成模块是系统的核心业务单元，集成了基于深度学习的多种图像处理能力。模块内部封装了统一的生成与编辑服务，对外暴露七个功能接口，分别对应文生图、指令编辑、局部重绘、背景生成、风格重绘及涂鸦作画六种模式以及一个模型工厂。在文生图功能中，用户可以灵活选择千问2.0 Pro、万象Pro、通义万相或本地模拟等多种模型后端，并通过尺寸、数量、负面提示词及随机种子等参数精细控制生成结果。编辑类功能则允许用户上传原始图像并结合掩码、风格标签或自然语言指令进行局部或全局的再创作。风格重绘功能提供12种艺术风格供用户选择。所有生成资源均通过 DashScope API 调用，返回结果在持久化存储后即时呈现给用户。该模块接口汇总于表3-3。

表3-3 图像生成模块接口

| 功能 | 端点 | 方法 | 说明 |
|------|------|------|------|
| 文生图 | /api/generate | POST | 根据文本提示词生成图像，支持批量生成（1-4张） |
| 指令编辑 | /api/edit | POST | 通过文字指令对图像内容进行修改 |
| 局部重绘 | /api/inpaint | POST | 在涂抹区域依据描述重新生成内容 |
| 背景生成 | /api/background | POST | 为透明背景主体合成新的背景 |
| 风格重绘 | /api/style-repaint | POST | 将人像转换为指定艺术风格（12种） |
| 涂鸦作画 | /api/sketch | POST | 将粗糙涂鸦完善为完整图像 |

3.3.4 历史记录模块

历史记录模块负责将用户每一次生成或编辑操作的请求参数与结果图像进行持久化记录，并提供回溯与管理功能。该模块通过关系型数据库中的 ImageRecord 表关联用户身份，完整保存图像类型、提示词、模型选择、尺寸参数、负面提示词、随机种子、生成数量、原始图像与结果图像的二进制数据等关键信息。用户可通过列表接口以分页形式浏览个人历史，并对不需要的记录进行删除。接口定义见表3-4。

表3-4 历史记录模块接口

| 功能 | 端点 | 方法 | 说明 |
|------|------|------|------|
| 历史列表 | /api/history | GET | 返回当前用户的生成编辑历史列表（分页） |
| 删除记录 | /api/history/<id> | DELETE | 删除指定历史记录 |

3.3.5 前端页面模块

前端页面模块基于 Vue.js 框架构建，采用组件化与路由管理的方式将各功能组织为四个独立视图，分别与后端业务模块相对应。LoginView.vue 负责登录与注册的交互，集成表单验证与状态提示；MainView.vue 是系统的操作主界面，整合图像生成与六种编辑功能的参数配置面板、结果预览区域以及实时调用逻辑，并支持深色/浅色主题切换；HistoryView.vue 提供个人历史记录的网格展示，支持查看大图、删除与下载；AdminView.vue 则面向管理员，呈现用户管理表格、历史查看弹窗与分页功能。各页面通过 Axios 与后端 API 通信，状态管理清晰，用户体验流畅。页面文件及功能对照见表3-5。

表3-5 前端页面模块

| 页面 | 文件 | 功能 |
|------|------|------|
| 登录页 | LoginView.vue | 用户登录与注册 |
| 主界面 | MainView.vue | 图像生成与编辑操作，深色/浅色主题切换 |
| 历史页 | HistoryView.vue | 个人历史记录浏览与管理 |
| 管理页 | AdminView.vue | 用户管理、查看用户历史、分页浏览 |
